P5 Execution & monitoringExecutionOrchestrator CGPT-4o COMPLEX_REASONING. Real-time decision making during multi-hour execution — anomaly classification, breakpoint detection, halt decisions. Rcreate_jira_defect — auto-creates [PERF]-prefixed defects with perf-defect label for CRITICAL and ERROR anomalies detected during execution. Cget_dynatrace_metrics + check_active_problems — primary real-time monitoring source. Polled every 60s. Problem entities signal breakpoint. Cstart_lre_test, check_lre_run_status, get_lre_results. Session-based auth (LWSSO_COOKIE_KEY). Polls every 30s with 5hr timeout. Ctrigger_jenkins_job — fires JMeter test plans. Returns queue item ID from Location header. Required for all non-VuGen scripts. CCPU, memory, JVM heap, GC pause, MQ depth, pod count via PromQL. Infrastructure health — catches resource exhaustion before it hits response times. Rsearch_error_logs — scans for OutOfMemoryError, Connection refused, Circuit breaker, 503, FATAL patterns. 5-min window per poll. CReserved LRE controller slot for LoadRunner tests. Shared resource across AT&T testing teams — must be scheduled in advance.

Cross-cutting & pre-pipeline dependencies
ALL BaseAgent framework HITL approval gate between every phase (web dashboard or step-through API). PipelineState persisted to disk as pipeline_state.json between phases. FastAPI + WebSocket backend must be running for dashboard and real-time status. Azure AD authentication for all Azure services (OpenAI, AI Search, AKS, App Gateway, SharePoint).
PRE Setup & governance AI governance: Azure OpenAI tenant approval within AT&T Azure environment; ML/AI Standards Team review of prompt templates and data handling practices. Snowflake: PERF_TESTING database provisioned via ddl_complete.sql (15 tables + 4 views with clustering keys). RAG index: Azure AI Search enterprise-knowledge index populated by Microsoft Graph connectors crawling SharePoint, PowerBI, ServiceNow, and Confluence. Env references: Per-application YAML files authored and manifest.yaml populated for all 22 in-scope applications (26 entries including multi-lab configs). App configs: Per-app .md files in templates/app_configs/ for Phase 2's read_app_config tool.

Key asymmetries

Phase 5 is the dependency bottleneck — 6 critical-path dependencies (Azure OpenAI, Dynatrace, LRE, Jenkins, Prometheus, LRE slot). Any one going down during the multi-hour execution window risks the entire test run.

RAG access is asymmetric — Phases 1, 2, 6 read from Azure AI Search. Only Phase 7 writes. Write credentials aren't needed until the pipeline's final phase.

Phase 3 owns infrastructure validation — the only phase touching AKS/kubectl, App Gateway, and Solace SEMP directly. All other phases interact with applications, not infrastructure.

CSTEM coordination is a human dependency — Phase 3 detects misconfigurations in minutes; remediation by CSTEM can take hours or days depending on their queue. This is typically the longest wall-clock delay in the pipeline.

Snowflake usage shifts across phases — Phases 1/2 read baselines, Phase 3 writes golden config, Phase 6 writes execution results (feeding future baselines), Phase 7 writes postmortem data. The same database serves both operational and analytical workloads.
